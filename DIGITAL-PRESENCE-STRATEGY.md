## 4. Google Analytics & Search Console

### Current State
The site already has GA4 partially configured:
- `BaseLayout.astro` conditionally loads the GA4 script using `PUBLIC_GA_MEASUREMENT_ID` env variable
- CTA click tracking is already implemented (tracks /contact and /intake clicks)

- Form submission tracking is implemented for `contact-form` and `lead-magnet-form`

### GA4 Setup Checklist

**Step 1: Create GA4 Property (if not done)**
1. Go to https://analytics.google.com/
2. Admin > Create Property
3. Property name: **RelayLaunch — relaylaunch.com**
4. Time zone: **Eastern Time (US & Canada)**
5. Currency: **US Dollar (USD)**

**Step 2: Create Web Data Stream**
1. Admin > Data Streams > Add Stream > Web
2. URL: `https://www.relaylaunch.com`
3. Stream name: **RelayLaunch Website**
4. Enable Enhanced Measurement (auto-tracks: page views, scrolls, outbound clicks, site search, video engagement, file downloads)
5. Copy the Measurement ID (format: G-XXXXXXXXXX)

**Step 3: Add Measurement ID to Environment**
Set the environment variable in your Cloudflare Pages environment:
```
PUBLIC_GA_MEASUREMENT_ID=G-XXXXXXXXXX
```
The BaseLayout.astro already conditionally loads GA4 in production using this variable.

**Step 4: Verify Installation**
1. Visit the live site
2. In GA4, go to Reports > Realtime
3. Confirm your visit appears
4. Also use GA4 DebugView: Admin > DebugView (install the Google Analytics Debugger Chrome extension)

**Step 5: Mark Key Events as Conversions**
In GA4 Admin > Events, mark these as key events (conversions):
1. `cta_click` — already fires when users click /contact or /intake links
2. `contact_form_submit` — already fires on contact form submission
3. `lead_magnet_submit` — already fires on lead magnet form submission
4. `lead_magnet_download` — already fires on /lead-magnets/ link clicks

**Step 6: Set Up Additional Events**
Add tracking for these events (add to the existing script block in BaseLayout.astro):

```javascript
// Track mailto clicks (contact page)
if (href.startsWith('mailto:')) {
  gtag('event', 'mailto_click', {
    event_category: 'engagement',
    event_label: href,
  });
}

// Track outbound GitHub clicks
if (href.includes('github.com/Relay-Launch')) {
  gtag('event', 'github_click', {
    event_category: 'engagement',
    event_label: href,
  });
}

// Track intake form page visit (high-intent signal)
// This is handled by GA4's automatic page_view event — create an audience for /intake visitors
```

**Step 7: Set Up Audiences**
In GA4, create these audiences for analysis:
1. **High-Intent Visitors** — Visited /contact OR /intake OR /complete-analysis
2. **Blog Readers** — Visited any /blog/* page
3. **Services Page Visitors** — Visited /services
4. **Returning Visitors** — Session count > 1

**Step 8: Enable Google Signals**
Admin > Data Settings > Data Collection > Turn on Google Signals
This enables cross-device tracking and demographic data.
